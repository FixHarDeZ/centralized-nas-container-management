## Architecture

### Source abstraction

Each source is a module in `sources/` directory with a standard interface:

```python
# sources/base.py
class Source:
    name: str           # e.g. "doujinth" or "dojintplthai"
    base_url: str       # full URL prefix
    
    def parse_listing(self, html: str) -> list[dict]
    def parse_title_page(self, html: str) -> dict
    def listing_url(self, page: int = 1) -> str
```

Each source returns items with keys: `slug`, `title`, `url`.

### Slug namespacing

Slug format: `<source_prefix>-<original_slug>` — stored in DB as `slug TEXT UNIQUE`.
- `doujinth` source: slug = `dt-<topic_id>` (e.g. `dt-111`)
- `dojintplthai` source: slug = `dc-<url_slug>` (e.g. `dc-แอบดูแชทแฟน-สุดแสนสะเทือนใจ`)

### Source tracking

- New `source TEXT` column on `titles` table (default `'doujinth'` for backward compat)
- `scrape_log` table gets `source TEXT` column
- New `db.source_stats()` returns per-source `{name: {count, size, last_scrape}}`
