# Memory Protocol

## 1. Directory Structure
- ทุก Stack ต้องมีโฟลเดอร์ `.notes/`
- ข้อมูลหลักอยู่ที่ `00_INDEX.md` (Blueprint) และ `daily_log.md` (Raw logs)

## 2. Note Content Format
เมื่อสรุปงาน ให้ใช้โครงสร้างนี้เสมอ:

### Session Log Entry
**Timestamp:** [YYYY-MM-DD HH:mm]
**Title:** [Short Descriptive Title]
**Details:**
- [Task performed 1]
- [Technical decision made]
- [Next steps/Pending]

## 3. Post-Task Action
- หลังแก้ไข Code สำเร็จ ให้เขียนสรุปลง `.notes/daily_log.md`
