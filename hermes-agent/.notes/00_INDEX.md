## Architecture

Three containers in the stack:
- **hermes-gateway** вЂ” from `hermes-agent` image, runs `hermes gateway run`; connects Telegram + Discord, processes messages
- **hermes-dashboard** вЂ” from `hermes-agent` image, runs `hermes dashboard`; listens only on internal port `9119`
- **hermes-nginx** вЂ” `nginx:alpine` sidecar; exposes `5063`, enforces HTTP Basic Auth via `.htpasswd`, proxies to `hermes-dashboard:9119`

Persistent volume: `hermes_agent_data:/opt/data` вЂ” config, sessions, memories, skills, logs

### Critical: HERMES_HOME

`HERMES_HOME=/opt/data` must be set in docker-compose.yml environment for both containers.  
Without it, hermes falls back to `~/.hermes` (ephemeral, not the volume) and loses config on restart.

### Model + Tune Config (config.yaml)

Live `/opt/data/config.yaml` is generated by hermes (full schema dump, ~13.8 KB). The repo's `config.yaml.example` documents the minimal **overrides** we apply on top вЂ” model selection plus the 2026-06-02 token tune.

`model:` is a mapping pointing to provider details:

```yaml
model:
  default: "mimo-v2.5-pro"     # current production; locked by user choice
  provider: "xiaomi"
  base_url: ""
```

Post-2026-06-02 token tune (verified keys in `hermes-v2026.5.16-schema.md`):

```yaml
session_reset:
  mode: both
  idle_minutes: 15             # was 1440 default вЂ” zombie sessions broke prefix cache

agent:
  max_turns: 20                # was 60 вЂ” cap iteration runaway
  api_max_retries: 1           # was 3 вЂ” faster failover
  image_input_mode: text       # was auto вЂ” pre-analyze images, kill vision tokens

memory:
  memory_enabled: false        # was true вЂ” 1-week trial; primary cache fix
  user_profile_enabled: false  # was true вЂ” same mechanism

compression:
  threshold: 0.80              # was 0.50 вЂ” delay context rewrite
```

## Known Gotchas

- **`HERMES_HOME` must be in docker-compose env** вЂ” entrypoint sets it locally but doesn't export to child `hermes` binary; falls back to `~/.hermes` (ephemeral) without explicit env
- **`model:` must be a mapping** вЂ” setting `model: <slug>` as scalar + indented sub-keys (default, provider) is invalid YAML; causes "No models provided" HTTP 400 from OpenRouter
- **Session reset key is `session_reset.idle_minutes`** вЂ” NOT `session.reset_on_idle_minutes` (a guess that hermes silently ignores). Default 1440 (24h) means sessions persist all day unless overridden. See `hermes-v2026.5.16-schema.md` for full key list.
- **Memory keys are `memory.memory_enabled` + `memory.user_profile_enabled`** вЂ” both inject mutable content into system prompt every turn, destroying upstream prefix cache when on. Off for 1-week trial through 2026-06-09.
- **`docker compose restart` can leave orphan s6-log lock** on the volume в†’ next start crash-loops with `s6-log: fatal: unable to lock /opt/data/logs/gateways/default/lock: Resource busy`. Fix: full stack `down` + `up -d` releases the lock.
- **Telegram logs success at INFO level** вЂ” only failures appear as WARNING/ERROR in docker logs; silence = success
- **Discord error is expected** вЂ” `ERROR: No bot token configured` is normal if Discord not used
- **`.htpasswd` permissions matter** вЂ” keep `nginx/.htpasswd` readable by nginx worker (`644`); deploy script already normalizes this on NAS
- **Old sessions store model per-session** вЂ” sessions created before config fix have empty model; new sessions use config
